# CSV Lead Enrichment Pipeline - Cursor Rules

## Project Context
Building a Next.js application that accepts a CSV of companies, enriches each using:
1. Website scraping (Firecrawl)
2. Search results (Serper)
3. Recent news (NewsAPI)
4. AI processing (Claude - two-step pipeline)
5. Email delivery (Resend)

## Stack
- Framework: Next.js 14 (App Router, TypeScript)
- Deployment: Vercel (60s timeout constraint)
- Processing: Promise.all() for 10 companies in parallel
- AI: Claude Sonnet 4 with structured JSON outputs (Zod)

## Key Constraints
- Vercel 60s timeout: must use Promise.all() for parallelism
- CSV parsing: Papaparse for input/output
- Error handling: fail gracefully per-row, never abort batch
- Two AI calls per company (not one): extraction → insights

## Folder Structure


app/
page.tsx              → Upload UI
api/
enrich/
route.ts          → Main POST endpoint
lib/
scrape.ts             → Firecrawl logic
search.ts             → Serper logic
news.ts               → NewsAPI logic
ai.ts                 → Claude Call 1 & Call 2
email.ts              → Resend logic
enrichRow.ts          → Orchestrates one company

## Code Style
- Use TypeScript strictly (no `any` unless unavoidable)
- Zod for schema validation
- try/catch error handling in every async function
- Environment variables: use process.env with optional chaining
- Comments only for "why", not "what"
